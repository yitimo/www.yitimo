# PuTsangTo 设计文档
## *更细历史*
* 无
## 一、 愿景
将打造一个3D平台冒险游戏，加入社交形式，玩家扮演冒险家逐步提升自己。
冒险的形式主要为平台跳跃、与地图元素互动、消灭怪物等，后期还可衍生更多丰富形式。
按键尽可能简单，类似街机连招，预期仅使用方向键以及五个以内动作键。
冒险分为常规地图冒险(多人在线)，以及副本地下城探险(单人或组队)。
故事线为海洋探险，分为各种类型海岛以及海域冒险。
考虑添加玩家专属岛屿，玩家可以建设自己的岛屿，进行生产、训练以及拜访等玩法。
支持两种视角操作：

1. 鼠标控制方向键盘移动，锁定第三人称视角(剑灵)
2. 侧面俯视，经典日式RPG视角3D倾斜版，参考冒险岛2(考虑添加此视角下的鼠标控制(英雄联盟))

常规地图为简单的平台跳跃以及打野怪，完成主线以及支线跑图收集任务，副本玩法使用以下几种：

1. 同常规地图但怪物以及地形难度增加，并添加一定目的性。
2. 潜入与平台跳跃，高难度的跑图收集。
3. 组队嘉年华，多玩家配合完成小游戏。
4. 大型Boss战斗，单房间打Boss。

玩家可选择几个基本职业，几个种族
形象可进行装饰，可捏脸改体型
游戏使用自己的独立时间。
### 1.1 游戏总结
海岛探险为故事线的3D平台跳跃RPG冒险游戏。

## 二、 受众、平台和市场
PC平台起步，为主打平台，后期考虑接入多端(网页端接入部分社交元素，移动端接入部分冒险元素)

配置尽量压低，主打玩法，考虑像素风，但不打算像冒险岛2做的那么极致。

参考现有游戏：

1. 冒险岛2
2. 魔兽3(RPG成分)

## 三、游戏玩法
### 3.1 核心玩法
玩家控制单角色，进行海岛地图探险，与地图元素互动以及消灭怪物完成特定目标，逐渐强化角色。

### 3.2 玩法描述
1. 玩家创建角色
2. 进入初始剧情(考虑根据不同职业分区域和故事线)
3. 教学关卡
4. 主线任务引导玩家
5. 不打算设立支线副职业任务，考虑全都整合进主线
6. 只设立里程碑副本，比如每个区域设立一个大型副本，其余均为野外区域(类似剑灵)

### 3.3 操作
1. GUI
角色状态信息放在中间下方(本人在剑灵中的习惯)，左下角展示消息，右下角为菜单，上方面板展示交互信息(战斗、怪物与其他玩家)，正上方展示特定消息

2. 操作规则
    WSAD移动，QE辅助移动或其他用途，鼠标方向，左右键动作，空格键跳跃，F键互动(FPS形式)
    
    方向键移动，alt跳跃，ctrl/shift动作，空格键互动，z/x/c辅助其他用途(冒险岛版RPG形式)

    WSAD移动，空格/j/k/l/m/n/u/i完善其他操作

互动的几种类型：
1. 触发环境元素(开关之类)
2. 抓起环境元素、释放/使用环境元素(参考松鼠大作战)
3. 对话交谈(NPC)
4. 拾取物品
5. 被动触发(特定地形、时间等)

### 3.4 模式和其他功能
初始阶段考虑包含以下模式：
1. 野外多玩家与地图互动(探索/任务/打怪与野外Boss)
2. 单人地下城闯关
3. 多人组队地下城闯关
4. 玩家间竞技(参考《冒险岛online》的怪物嘉年华)
5. 暂不考虑加入纯pvp，纯属因为个人讨厌pk
### 3.5 关卡
在每个模式下的关卡玩法分为以下几种：
1. 地图探险跳跃(《冒险岛online》中的跳跃任务)
2. 组队任务(《冒险岛online》中的组队任务)
3. Boss地下城战斗(DNF)
4. 怪物嘉年华(大量怪物，累计分数(《冒险岛online》怪物嘉年华、DNF各种刷塔))
5. 世界Boss、大型Boss、野外Boss战斗
* 所有关卡都基于平台设计，考虑重力、地形以及可互动建筑
* 拒绝太过像素风，但底层仍选择像素风
### 3.6 流程表
* 启动阶段
    1. 登录认证流程
    2. 服务器选择
    3. 角色选择
    4. 每个地图设立多个固定出生点(无论上次从何处下线，都将出生在最后一次停留的野外地图的固定出生点中)
* 冒险阶段
    1. 地图之间切换
    2. 完成收集/消灭/交谈等任务
    3. 野外Boss
    4. 进入特定房间(副本等待等)
* 关卡阶段
    1. 地下城的入口处地图/任务等待房间
    2. 召集队友等
    3. 生成副本
    4. 多玩家进入
    5. 玩家离开后副本将被回收(多人副本则为所有玩家离开后才回收)
    6. 完成副本的任务
    7. 离开副本回到最后一次停留的野外地图或房间
* 下线阶段
    1. 离线后角色从世界消失，保存状态
    2. 记录最后一次停留的野外地图(包括未回收的队伍、副本状态)

## 四、 游戏角色
*初步版本暂不考虑角色形象，只考虑功能*
### 4.1 角色设计
采用转职体系：
1. 战士(近战、肉盾、笨重) -> 圣骑士(转魔法、灵活、脆皮)/英雄(极致肉盾)
2. 刺客(近战、输出形式独特、脆皮、灵活) -> 忍者(转远程、灵活)/武士(近战、均衡)
3. 魔法师(远程、爆发、脆皮、笨重) -> 战斗法师(近战、元素、灵活)/元素师(极致元素)
4. 射手(远程、笨重、伤害稳定高、脆皮) -> 狙击手(笨重、极致伤害)/猎人(近战加远程、均衡)

### 基本操作
#### 上下左右移动
由于地图为侧面倾斜：
* 上为左上
* 下为右下
* 左为左下
* 右为右上
* 跳跃与下蹲，配合移动方向为方向跳跃与潜行
#### 其他动作
* 互动，触发场景、NPC对话
* 加速与闪避
* 攻击、防守与技能
